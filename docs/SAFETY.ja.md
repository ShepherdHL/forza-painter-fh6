# 安全ガイド（Forza Horizon 6）

この文書は forza-painter FH6 が PC 上で行うことと、リスクを抑える使い方を説明します。

## インポート前

1. **Vinyl Group Editor に留まる** — FH6 で「Create Vinyl Group」/「Vinyl Group Editor」を開き、インポート中はメニューを切り替えない。
2. **テンプレートのグループ解除** — グループ化されたビニールはレイヤー書き込みに失敗します。
3. **レイヤー数を確認** — アプリにゲーム内テンプレートの正確なレイヤー数を入力（FH6 必須）。
4. **適合する JSON** — 低レイヤー JSON を高レイヤーテンプレートに入れるとぼやけます。

## インポートをクリックしたとき

1. **同意** ダイアログ：対象プロセスと、ビニールエディターデータのみ変更する旨。
2. **管理者** 権限がない場合、**管理者として再起動** を求めます（UAC 1 回）。FH6 への `OpenProcess` 用で、オフライン生成には不要です。
3. **自動ロケート** で FH6 レイヤーテーブルを検索（読み取り）し、検証後に書き込みます。
4. **ヘルパー**（同一アプリの helper モード）がインポートを実行。ログにどのヘルパーか表示されます。

## 変更しないもの

速度、クレジット、車両、レース記録、オンライン統計などは変更しません。エディターのビニールレイヤーフィールドのみです。

## アカウント / BAN リスク

FH6 実行中のメモリ操作は外部メモリツールと同種です。見た目のみでも、ポリシーまたはアンチチートの対象になる可能性があります。

**自己責任でご利用ください。**

## 推奨ワークフロー

| 手順 | ゲーム起動？ | 管理者？ |
| --- | --- | --- |
| JSON 生成（GPU） | 任意（FH6 終了で GPU 温度低減） | いいえ |
| プレビュー / フィルター | いいえ | いいえ |
| FH6 へインポート | **はい**（エディター） | **はい**（求められた場合） |
| FH6 からエクスポート | **はい** | **はい**（求められた場合） |

## ツール → FH6 診断

スナップショット、比較、テーブル検査はメモリをスキャンします。**メモリデバッグを理解していない限り使用しないでください。** 通常のインポートには不要です。

## 「アクセス拒否」のとき

1. 昇格プロンプトを受け入れ、管理者として再起動。
2. 再度インポート。
3. FH6 をロックしている他のメモリツールを終了。
4. Vinyl Group Editor にいること、PID が正しいことを確認。

## experiments フォルダ

`experiments/` のスクリプトは**日常利用非推奨**で、FH6 がクラッシュする場合があります。通常のインポートフローを使用してください。
