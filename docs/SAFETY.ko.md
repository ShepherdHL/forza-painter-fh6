# 안전 가이드 (Forza Horizon 6)

이 문서는 forza-painter FH6가 PC에서 하는 일과 위험을 줄이는 사용법을 설명합니다.

## 가져오기 전

1. **Vinyl Group Editor 유지** — FH6에서 «Create Vinyl Group» / «Vinyl Group Editor»를 연 채로 가져오기를 진행하세요.
2. **템플릿 그룹 해제** — 그룹화된 비닐은 레이어 쓰기에 실패할 수 있습니다.
3. **레이어 수 확인** — 앱에 게임 내 템플릿의 정확한 레이어 수를 입력하세요 (FH6 필수).
4. **맞는 JSON** — 낮은 레이어 JSON을 높은 레이어 템플릿에 넣으면 흐릿해집니다.

## 가져오기를 클릭하면

1. **동의** 요약: 대상 프로세스, 비닐 편집기 데이터만 수정.
2. **관리자** 권한이 없으면 **관리자로 다시 시작** (UAC 1회). FH6 `OpenProcess`용이며 오프라인 생성에는 불필요합니다.
3. **자동 찾기**로 FH6 레이어 테이블을 읽고 검증 후 씁니다.
4. **헬퍼**(동일 앱 helper 모드)가 가져오기를 실행합니다. 로그에 헬퍼 이름이 표시됩니다.

## 변경하지 않는 것

속도, 크레딧, 차량, 기록, 온라인 통계 등은 변경하지 않습니다. 편집기 비닐 레이어 필드만 수정합니다.

## 계정 / 제재 위험

FH6 실행 중 메모리 읽기/쓰기는 외부 메모리 도구와 유사합니다. 외형만 바꿔도 정책 또는 안티치트 대상이 될 수 있습니다.

**본인 책임으로 사용하세요.**

## 권장 워크플로

| 단계 | 게임 실행? | 관리자? |
| --- | --- | --- |
| JSON 생성 (GPU) | 선택 (FH6 종료 시 GPU 온도↓) | 아니오 |
| 미리보기 / 필터 | 아니오 | 아니오 |
| FH6 가져오기 | **예** (편집기) | **예** (요청 시) |
| FH6보내기 | **예** | **예** (요청 시) |

## 도구 → FH6 진단

스냅샷, 비교, 테이블 검사는 메모리를 스캔합니다. **메모리 디버깅을 이해하지 않으면 사용하지 마세요.** 일반 가져오기에는 필요 없습니다.

## «액세스 거부» 오류 시

1. 권한 상승 후 관리자로 다시 시작.
2. 가져오기 재시도.
3. FH6 프로세스를 잠그는 다른 메모리 도구 종료.
4. Vinyl Group Editor 및 PID 확인.

## experiments 폴더

`experiments/` 스크립트는 **일상 사용 비권장**이며 FH6이 충돌할 수 있습니다. 기본 가져오기 흐름을 사용하세요.
